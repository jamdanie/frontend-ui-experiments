# Frontend UI Experiments

A curated collection of small front-end studies, layout labs, and visual interface experiments built while learning and refining HTML, CSS, and JavaScript fundamentals.

This repository groups together several smaller projects that focus on:

- layout structure
- visual hierarchy
- responsive composition
- motion and interaction
- aesthetic experimentation
- front-end practice without heavy frameworks

These projects are not intended to be production applications.  
They are learning-focused builds and design studies that document front-end growth over time.

---

## Purpose

This repo exists to keep smaller UI and layout projects in one organized place rather than scattering many tiny repositories across GitHub.

Each folder contains its own self-contained project with its own files and local README.

---

## Included Projects

### `dark-premium-ui`
A modern one-page portfolio concept built with plain HTML, CSS, and JavaScript.

Focus areas:
- clean portfolio structure
- dynamic project rendering
- lightweight filtering
- polished visual presentation

### `layout-practice`
A pure HTML/CSS layout study focused on spacing, structure, layering, and visual hierarchy.

Focus areas:
- responsive layout fundamentals
- CSS-only visual depth
- composition without JavaScript
- typography and spacing discipline

### `mecha-ui-concept`
A single-page UI study inspired by futuristic mecha / anime design language.

Focus areas:
- hero-driven composition
- layered UI treatment
- subtle motion design
- responsive presentation
- experimental visual styling

### `ridge-relay-ui-prototype`
An early front-end prototype related to the RidgeRelay concept, focused on interface structure and wireframe thinking.

Focus areas:
- UI flow exploration
- product concept presentation
- layout and interaction planning
- early design experimentation

> Note: the primary RidgeRelay project is maintained separately as its own main repository.  
> This version is included here only as an earlier UI/prototype study.
